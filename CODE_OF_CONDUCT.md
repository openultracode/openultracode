# Code Of Conduct

OpenUltraCode is an open-source project for local, auditable coding-agent workflows. Contributors are expected to keep discussion practical, respectful, and focused on improving the project.

## Expected Behavior

- Be direct without being dismissive.
- Assume good intent while still checking technical claims.
- Keep criticism tied to code, docs, tests, designs, or project decisions.
- Share enough context for others to reproduce issues.
- Respect security and privacy boundaries, especially around secrets, logs, local run artifacts, and model outputs.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Discriminatory language or exclusionary conduct.
- Publishing private information without permission.
- Pressuring contributors to disclose secrets, tokens, personal data, or private logs.
- Intentionally submitting unsafe instructions, prompt-injection content, or destructive commands as if they were trusted project guidance.
- Repeatedly derailing issues or reviews after maintainers ask for a narrower scope.

## Scope

This code applies in project spaces including GitHub issues, pull requests, discussions, security reports, and any shared project chat or review surface.

## Reporting

For normal conduct issues, open a GitHub issue or contact a maintainer through the repo.

For sensitive reports that involve security, secrets, private logs, or exploit details, use the private reporting path in `SECURITY.md`.

## Enforcement

Maintainers may edit, hide, or remove comments, close issues, decline contributions, or restrict participation when behavior is harmful to the project or its contributors.

Enforcement should be proportional, documented where practical, and focused on protecting useful collaboration.
